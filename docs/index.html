Introduction

WiFi-PWN Toolkit is a comprehensive, offensive WiFi security assessment framework built in Python. It automates the full attack lifecycle — from reconnaissance to password recovery — using built-in or external WiFi adapters.

Designed for penetration testers and security researchers, it supports both Windows and Linux, and automatically installs missing dependencies on Linux.

Features

Saved Password Extraction

Extracts all saved WiFi passwords from Windows (netsh) or Linux (NetworkManager / wpa_supplicant). No capture needed.

PMKID Capture

Captures PMKID hashes directly from APs using hcxdumptool. Works on most internal WiFi adapters without monitor mode.

WPA Handshake Capture

Classic 4-way handshake capture via airodump-ng in monitor mode. Requires an adapter that supports injection.

Deauth Attacks

Sends deauthentication frames via aireplay-ng to force client reconnections and trigger handshakes.

Password Cracking

Crack PMKID or handshake hashes with hashcat (mode 22000). Supports dictionary, mask, and rule-based attacks.

Bundled Wordlist

Includes rockyou.txt (14M+ passwords) so attacks work out-of-the-box. No downloads required.

Auto-Install Tools

On Linux, missing tools are detected automatically. The script prompts: "Install missing tools via apt? [Y/n]"

Auto Scan & Full Mode

scan lists nearby networks. full runs recon → capture → crack in one command.

How It Works

The toolkit follows a modular, phase-based attack methodology. Each phase is independent and can be run standalone or as part of the full automation chain.

1

Reconnaissance (recon)

Gathers intelligence locally by reading saved WiFi profiles from the operating system. On Windows, it queries netsh wlan show profiles. On Linux, it reads /etc/NetworkManager/system-connections/ and /etc/wpa_supplicant/. This recovers plaintext passwords for previously connected networks instantly.

2

PMKID Capture (pmkid)

Uses hcxdumptool to send association requests to nearby access points. If the AP supports PMKID (most modern routers do), it returns a PMKID hash. This is a client-less attack — no client needs to be connected. The hash is saved as a .22000 file compatible with hashcat.

3

Handshake Capture (capture)

Enables monitor mode using airmon-ng, then uses airodump-ng to listen for WPA 4-way handshakes. A handshake occurs whenever a client connects to the target AP. Optionally, a deauth attack can be sent first to force a reconnection.

4

Deauth Attack (deauth)

Sends crafted 802.11 deauthentication frames to a specific client or broadcast. This forces the client to disconnect and reconnect, triggering a new WPA handshake that can then be captured.

5

Conversion (hcxpcapngtool)

Raw capture files (.pcapng / .cap) are converted into hashcat format (.22000) using hcxpcapngtool. This extracts the PMKID or handshake hash into a crackable format.

6

Cracking (crack / full)

Passes the .22000 hash file to hashcat along with a wordlist. The toolkit includes rockyou.txt (14M+ real-world passwords) for immediate use. Supports dictionary attacks, mask attacks, and rule-based mangling.

7

Reporting

All results — cracked passwords, SSIDs, BSSIDs, timestamps, and cracking speeds — are saved to audit-report.txt and cracked-passwords.txt automatically.

Requirements

Linux (Recommended)

  • Python 3.8 or higher
  • Root / sudo privileges
  • Wireless adapter (internal or external) with monitor mode support

Dependencies: aircrack-ng, hcxdumptool, hcxtools, hashcat, iw, ethtool

Windows

  • Python 3.8 or higher
  • Administrator privileges
  • WiFi adapter (built-in or USB)

Note: On Windows, only recon, scan, and detect are natively supported. For PMKID/handshake capture, use a Linux environment or WSL2 with a compatible adapter.

Installation

Clone the Repository

git clone https://github.com/jojin1709/wifi-pwn-toolkit.git
cd wifi-pwn-toolkit

Install Dependencies (Linux)

sudo apt-get update
sudo apt-get install -y aircrack-ng hcxdumptool hcxtools hashcat python3 iw ethtool

Or let the toolkit auto-install them — just run any command and press Y when prompted.

Run

# Linux
sudo python3 wifi-pwn.py detect

# Windows
python wifi-pwn.py --no-admin detect

Quick Start

1. Check Adapter Capabilities

sudo python3 wifi-pwn.py detect

2. Extract Saved WiFi Passwords

sudo python3 wifi-pwn.py recon

3. Scan Nearby Networks

sudo python3 wifi-pwn.py scan

4. Capture PMKID and Crack

sudo python3 wifi-pwn.py pmkid --bssid AA:BB:CC:DD:EE:FF --timeout 120
sudo python3 wifi-pwn.py crack --input wifi-pwn-output/pmkid-*.22000

5. Full Automation

sudo python3 wifi-pwn.py full --bssid AA:BB:CC:DD:EE:FF --timeout 120

Command Reference

recon — Extract Saved WiFi Passwords

Recover plaintext WiFi passwords stored on the local machine. Works instantly without any wireless traffic.

python3 wifi-pwn.py recon

OS sources:

  • Windows: netsh wlan show profiles + key=clear
  • Linux: /etc/NetworkManager/system-connections/, /etc/wpa_supplicant/wpa_supplicant.conf

Output: wifi-pwn-output/saved-passwords.txt

scan — Scan Nearby Networks

Lists all visible WiFi networks with SSID, BSSID, channel, signal strength, and encryption type.

python3 wifi-pwn.py scan
python3 wifi-pwn.py scan --interface wlan0

pmkid — Capture PMKID Hash

The PMKID attack is the most reliable method for internal/adapters. It requests a PMKID from the AP directly — no client traffic or monitor mode required.

python3 wifi-pwn.py pmkid
python3 wifi-pwn.py pmkid --bssid AA:BB:CC:DD:EE:FF --timeout 120

How PMKID Works

When a client requests association, the AP includes a PMKID (Pairwise Master Key ID) in its response. This PMKID is derived from the AP's PMK (which is the same as the WiFi password). If we capture it, hashcat can crack it offline.

Output: .pcapng capture → .22000 hash file

capture — WPA Handshake Capture

Captures the classic WPA 4-way handshake. Requires monitor mode and an adapter that supports packet injection.

python3 wifi-pwn.py capture --bssid AA:BB:CC:DD:EE:FF --channel 6
python3 wifi-pwn.py capture --bssid AA:BB:CC:DD:EE:FF --channel 6 --timeout 180

Requirements: Monitor mode (airmon-ng), injection-capable adapter

Tip: If monitor mode fails, use pmkid instead — it works on more adapters.

deauth — Deauthentication Attack

Sends deauth frames to disconnect a client or the entire network. Often used before capture to force a handshake.

# Broadcast deauth (disconnects all clients)
python3 wifi-pwn.py deauth --bssid AA:BB:CC:DD:EE:FF --iface wlan0mon --count 10

# Targeted deauth (specific client)
python3 wifi-pwn.py deauth --bssid AA:BB:CC:DD:EE:FF --client 11:22:33:44:55:66 --iface wlan0mon --count 5

crack — Password Cracking

Crack PMKID or handshake hashes using hashcat. Supports dictionary, mask, and rule attacks.

# Dictionary attack (uses bundled rockyou.txt by default)
python3 wifi-pwn.py crack --input hash.22000

# Custom wordlist
python3 wifi-pwn.py crack --input hash.22000 --wordlist /path/to/wordlist.txt

# Mask attack (e.g. 8-digit numeric PIN)
python3 wifi-pwn.py crack --input hash.22000 --mask "?d?d?d?d?d?d?d?d"

# With rules for mangling
python3 wifi-pwn.py crack --input hash.22000 --wordlist rockyou.txt --rules /usr/share/hashcat/rules/best64.rule

# Auto-convert .cap files
python3 wifi-pwn.py crack --input capture.cap

Hashcat Mode: 22000 (WPA-PBKDF2-PMKID+PMK / WPA-PMKID+PMK / WPA-PMKID+PMK-replay)

full — Automated End-to-End Mode

Runs the complete attack chain: recon → scan (optional) → PMKID capture → crack.

python3 wifi-pwn.py full --bssid AA:BB:CC:DD:EE:FF --timeout 120

If no BSSID is provided, it scans first and lists available targets.

detect — Check Adapter Capabilities

Diagnostic command that checks what your WiFi adapter supports before you attempt attacks.

python3 wifi-pwn.py detect

Reports:

  • Available wireless interfaces
  • Monitor mode support (via iw)
  • Driver information
  • PMKID capture feasibility
  • Installed tool status

Troubleshooting

"No wireless interfaces found"

Run iw dev to check available interfaces. Your adapter may need additional drivers, or may not support monitor mode.

"Monitor mode failed"

Switch to pmkid mode — it works on most internal adapters without monitor mode. Some virtual adapters (VMware, VirtualBox) do not support it.

"No PMKID captured"

Ensure the AP is within range (< 20m clear line-of-sight recommended). Try a longer timeout (--timeout 120). Not all APs send PMKID — older or low-end routers may omit it.

"Permission denied"

Linux: Always run with sudo. Windows: Run Command Prompt or PowerShell as Administrator.

Hashcat "No hashes loaded"

Ensure the .22000 file is valid. Check its size — it should be at least 100 bytes. Re-run hcxpcapngtool conversion if needed.

Slow cracking speed

Use --mask for targeted attacks instead of full dictionary. Ensure your GPU drivers are installed for hashcat. CPU-only cracking is very slow for large wordlists.
